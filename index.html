Big Idea 1 — Creative Development

In this unit, I learned to turn an idea into a working program by planning, testing, showing, and explaining my design. I took notes and screenshots as documentation and made a short narration to show how the program works and why I built it.

  • 1.1 Collaboration: We used teamwork to split tasks and problem solving to block each other to dont get bugs.
  • 1.2 Program Design & Development: I have developed simple algorithms, drafted pseudocode, and used debugging techniques such as print checks and step-by-step tests to help refine the behavior.
  • 1.3 Program Function & Purpose: Set a clear goal and identified the target audience, organized the input and output processes, grouped the logic into functions and procedures, and documented everything carefully so that others can easily understand and reuse the work.
  • 1.4 Identifying & Correcting Errors: Wrote small tests, tried edge cases, and added basic error handling to avoid crashes or invalid inputs incode.

Scratch Tic-Tac-Toe — Documentation

This Scratch project is a Tic-Tac-Toe game I built with help from my friend Ö.E.H. Nine tiles toggle between X and O; win sprites check rows, columns, and diagonals, highlight the winning line, and stop input. Next steps: fix edge-case bugs, add win/draw counters, a reset and scoreboard, player vs. CPU mode, sound effects, and cleaner code.
